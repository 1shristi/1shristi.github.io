All-in-One Shipping and Marketplace Solution

to replace a broken legacy system

Overview Small team | Lead designer | 4 weeks
The client had a comprehensive roadmap to replace an outdated, monolithic legacy system used for shipping international cosmetic goods, which had become a bottleneck over time. I was brought in to lead research and interface design, translating legacy flows into efficient UI for a permissioned B2B marketplace.
Research Process Details

Analysing the Application, Service Safari Method
SOPs guided my understanding of real-world interactions. I used the service safari method to observe and extract insights.

Discovery: Legacy system should be deprioritized—focus instead on user tasks and physical workflows.

Physical Documents as UI Seeds
Documents were used only when systems were unavailable—prone to errors and ripe for digitization.

Discovery: Physical docs = fallback tools, showing where UI must bridge gaps.

Ethnographic 1:1 Interviews
No interview script—just active listening. Learned that many daily tasks were inefficient workarounds.

Discovery: Workarounds were mistaken for essential tasks—revealed scope for smarter UI paths.

Sketching for agreement

There is nothing particularly unique about the prototyping process of sketching a solution and getting feedback. I frequently shared and discussed my findings and sketches with the PO. As the designer, I find these discussions helpful for aligning with the PO’s expectations on what features or pain points to address. The interface should not only align with user needs but also align with expectations of the PO's vision of the product.
