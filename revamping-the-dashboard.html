Demo application is ready but visually lacking

“When presenting screenshots of the application to potential buyers, the UI doesn’t sell all the features it is capable of” --Client

Without usability testing, the solution for “visually lacking” will be dangerously subjective. This is one of the four design constraints that I learned while scoping the project.

Good news? Constraints can tell us where to begin the design process.

4 constraints to designing a solution
  • Little qualitative and quantitative usability data to rely on
  • No time to arrange user interviews
  • Must be in scope of design system
  • Design team of one

Prioritizing the relevant content

The fewer the actions required by the user and the higher the priority of the task is for the user, the more useful it will be to display the content on the Dashboard.

Monolithic architecture

Borrowing language from programming, the remaining components from the matrix graph above can broadly be seen as monolithic, whereby one menu item (i.e. Space management) relies on the previous one (Site settings) in order to function.

New Dashboard should
  • Guide user around one area of control since all the other areas are too complex (i.e. monolithic architecture)
  • Find opportunity to use spatial assets as a visual component to the application
